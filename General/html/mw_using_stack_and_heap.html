栈，堆和 RTOS 配置

每个软件组件都有栈，堆和内存的资源需求。像 startup_device.s 和 RTX_Conf_CM.h 这样的配置文件需要适应这些要求。每个中间件组件都有一个解释资源需求的部分，并举例说明如何计算用于修改配置文件的结果值。

使用 startup_device.s 配置堆栈

startup_device.s 文件由设备系列包中的设备供应商提供。在 Manage Run-Time Environment 窗口中选择 Device:Startup 后，该文件将被自动复制到 μVision 项目中。它用于配置当前项目的异常和中断服务程序（ISR）使用的堆栈大小。堆栈大小要求取决于异常和 ISR 执行的最大嵌套，因此取决于各种中断和异常源的优先级设置。另外，在这个文件中配置内存分配函数使用的堆。

大多数中间件组件只需要额外的栈大小，而文件系统组件还需要额外的堆大小。内存需求在每个中间件组件的资源需求部分中说明。

使用 RTX_Conf_CM.c 配置 CMSIS-RTOS

startup_device.s 文件中的堆栈大小仅配置异常和中断服务程序（ISR）使用的堆栈空间。当使用 CMSIS-RTOS 时，需要在 RTX_Conf_CM.c 文件中配置 RTOS 的附加要求。根据中间件组件，例如需要配置用户线程数量，线程堆栈大小甚至用户计时器。

每个中间件组件的资源需求部分还介绍了额外的 CMSIS-RTOS 相关需求。

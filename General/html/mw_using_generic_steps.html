通用步骤

在 μVision 项目中包含一些中间件组件需要执行几个通用步骤：

将中间件添加到 μVision 项目的一般步骤

添加软件组件

使用 Manage Run-Time Environment 窗口，您需要为应用程序选择所需的软件组件。Validation Output 可帮助您识别其他软件组件的依赖关系，并尽可能自动解决它们。

使用验证错误管理运行时环境窗口

配置中间件

每个中间件组件都有特定的配置文件，需要根据应用程序的需要进行调整。根据组件的不同，可能需要修改多个文件。例如，File System 组件需要配置可以同时打开的文件数量和 FS_Config.c 文件中的初始当前驱动器。此外，您需要在 FS_Config_Drive_0.h 文件中设置驱动器特定设置，例如 Flash 存储器的页面和块大小，RAM 驱动器的常规磁盘大小以及用于将驱动器连接到相应的硬件外围驱动器的硬件驱动器编号。

文件系统配置文件 FS_Config_MC_0.h 和 FS_Config.c

配置驱动程序

通常，还需要配置与微控制器中的软件组件和相关控制器相关的设备驱动程序。对于中间件，这通常在文件 RTE_Device.h 中完成。在这里，您可以启用或禁用不同的硬件控制器，并且可以指定哪些器件引脚用于特定器件外设的不同功能。

用于驱动程序配置的 RTE_Device.h 文件

配置系统资源

每个软件组件都有栈，堆和内存的资源需求。像 startup_device.s 和 RTX_Conf_CM.h 这样的配置文件需要适应这些要求。每个中间件组件都有一个解释资源需求的部分，并举例说明如何计算用于修改配置文件的结果值。栈，堆和 RTOS 配置 部分为您提供了关于需要触及的背景和配置文件的更多信息。

用于系统资源配置的 RTX_Conf_CM.c 和 startup_device.s

实现应用程序特征

最后，需要实现实际应用。为了更好地理解中间件组件，请查看现有 Example projects。它们可以作为应用程序开发的起点。每个中间件组件的 参考 部分有助于理解相关组件的 API 函数。各种 可帮助您实现您的应用程序。
